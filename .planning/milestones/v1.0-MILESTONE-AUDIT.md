## Integration Gaps

### INT-04: $MILESTONE_NAME not parsed (P0 — Runtime Error)

**Affects:** AUTO-08 (end-of-run report)
**Location:** `skills/execute-phase/SKILL.md` line 37
**Issue:** The `initialize` step parses init JSON but omits `milestone_name` from the field list. The `offer_next` step calls `autonomous generate-report --milestone "$MILESTONE_NAME"`, but `$MILESTONE_NAME` is never assigned. `cmdGenerateReport` requires `--milestone` and calls `error()` if missing, causing a non-zero exit.
**Fix:** Add `milestone_name` to the execute-phase initialize step parse list.

### INT-05: execute-plan.md missing from plugin root (P1)

**Affects:** FOUND-03, FOUND-04, AUTO-04
**Location:** Referenced by `skills/execute-phase/SKILL.md` line 313, `agents/gsdr-planner.md` line 431
**Issue:** `@${CLAUDE_SKILL_DIR}/../execute-plan.md` resolves to nothing. Executor and planner agents run without this context document. Not a hard failure (Claude Code ignores missing `@file:` references) but degrades agent quality.
**Fix:** Port `~/.claude/get-shit-done/workflows/execute-plan.md` to plugin root.

### INT-06: transition.md missing from plugin root (P1)

**Affects:** AUTO-03, AUTO-04
**Location:** Referenced by `skills/execute-phase/SKILL.md` line 804, `skills/resume-work/SKILL.md` line 253
**Issue:** The auto-advance chain's phase transition step instructs "Read and follow transition.md" but the file doesn't exist. Without it, the `--auto` flag may not propagate to the next phase, breaking the autonomous chain.
**Fix:** Port GSD transition.md workflow to plugin root.

## Broken E2E Flows

### End-of-Run Report Generation (AUTO-08)

**Flow:** execute-phase -> offer_next -> autonomous generate-report -> END-OF-RUN-REPORT.md
**Breaks at:** `offer_next` step — `$MILESTONE_NAME` undefined
**Impact:** Every auto-advance chain completion or `--milestone` run will fail to produce the end-of-run report. The `error()` call exits the CLI tool non-zero, and the report file is never written.

### Auto-Advance Chain Transition (Degraded)

**Flow:** execute-phase -> transition.md -> next plan-phase with `--auto`
**Breaks at:** transition.md reference
**Impact:** The orchestrator improvises the transition without instruction. The `--auto` flag may or may not propagate. Not a hard failure but makes autonomous behavior non-deterministic.
