parseDependsOn(dependsOnStr: string | null): string[]
- If null/undefined, return []
- Normalize to lowercase, check for "nothing", "first phase", "none" -> return []
- Extract all phase numbers via regex /\d+(?:\.\d+)*/g
- Return unique array via Set

buildDependencyGraph(cwd: string): DependencyGraph
- Read ROADMAP.md directly from cwd/.planning/ROADMAP.md
- Parse phase headers and Depends on fields (reuse regex patterns from roadmap.ts: /\*\*Depends on:\*\*\s*([^\n]+)/i)
- For each phase, read plan frontmatter via extractFrontmatter() to get files_modified and wave
- Do NOT call cmdRoadmapAnalyze (it calls output()). Parse directly.
- Build PhaseNode[] and DependencyEdge[] arrays

computeBatches(graph: DependencyGraph, maxAgents?: number): ExecutionBatch[]
- Use Kahn's algorithm (BFS-based topological sort)
- Build in-degree map from edges
- Process nodes with in-degree 0 as a batch, then decrement dependents
- Detect cycles: if nodes remain after BFS completes, report cycle
- Skip already-complete phases
- Default maxAgents = 5, cap at 10

detectFileConflicts(graph: DependencyGraph, batchPhases: string[]): FileConflict[]

- Normalize paths: replace backslashes with forward slashes
- Plans with empty files_modified: treat as conflicting with ALL other plans in the batch (fail-safe). Create a synthetic conflict entry with file "__UNSAFE_NO_FILES_DECLARED__" listing the unsafe plan and all other plans.
- Build Map<string, string[]> of file -> planIds
- Return entries where planIds.length > 1, with resolution: 'sequential'

distributeAgentBudget(batch: ExecutionBatch, graph: DependencyGraph, maxAgents: number): AgentBudget[]
- Count total plans across all phases in batch
- If totalPlans <= maxAgents: each phase gets what it needs
- Otherwise: proportional allocation with minimum 1 per phase
- Adjust down if total exceeds budget (reduce largest allocation first)
- Total allocated MUST NOT exceed maxAgents
