  <behavior>

  ## normalizeErrorSignature(error: string): string
  - Strips line:col numbers (`:42:10` -> `:LINE:COL`)
  - Strips stack trace frames (`at foo (/path/file.ts:10:5)` -> `at STACK`)
  - Strips file paths (`/Users/foo/bar.ts` -> `FILE`)
  - Strips ISO timestamps -> `TIMESTAMP`
  - Strips hex addresses (`0x1a2b3c` -> `ADDR`)
  - Trims whitespace
  - Two errors with same root cause but different line numbers produce identical normalized signatures

  ## FailureEntry interface
  ```typescript
  interface FailureEntry {
    plan_id: string;
    status: 'active' | 'resolved' | 'halted' | 'skipped_upstream_failure';
    error_signature: string;
    attempts: AttemptRecord[];
    strike_count: number;
    halted_reason?: string;
    blocked_by?: string;
  }
  interface AttemptRecord {
    attempt: number;
    timestamp: string;
    error: string;
    diagnosis: string;
    solution_tried: string;
    result: 'resolved' | 'same_error' | 'no_improvement' | 'new_error';
  }
  ```

  ## appendFailure(failuresPath: string, entry: FailureEntry): void
  - Creates FAILURES.md with YAML frontmatter if it does not exist
  - Appends new entry section to existing FAILURES.md
  - Updates frontmatter counts (total_failures, resolved, halted, skipped)
  - Entry format: `## {plan_id}: {status}` with YAML-like body

  ## readFailures(failuresPath: string): { frontmatter: object, entries: FailureEntry[] }
  - Parses FAILURES.md frontmatter and all entry sections
  - Returns empty entries array if file does not exist
  - Handles malformed entries gracefully (skips, does not crash)

  ## updateFailureStatus(failuresPath: string, planId: string, status: string, attemptRecord?: AttemptRecord): void
  - Updates an existing entry's status field
  - Appends new attempt record if provided
  - Recalculates frontmatter counts

  ## isNonImproving(prevAttempt: AttemptRecord, currAttempt: AttemptRecord, normalizedPrevError: string, normalizedCurrError: string): boolean
  - Returns true if normalized error signatures match (same_error)
  - Returns true if currAttempt.result is 'no_improvement'
  - Returns false if error signature changed AND result is not 'no_improvement' (new_error with test improvement)
  - Per user decision: "same error signature OR no test improvement -- either counts as a strike"

  ## checkIrreversibleAction(actionContent: string, defaultPatterns: string[], configOverrides?: string[]): { matched: boolean, matches: string[] }
  - Scans action content for irreversible Bash command patterns
  - Merges defaultPatterns with configOverrides (overrides extend, not replace)
  - Returns matched pattern strings
  - Matches: `git push --force`, `git push -f`, `git reset --hard`, `npm publish`, `DROP TABLE`, `rm -rf /`, `git clean -f`, `git branch -D`, `docker system prune`, `kubectl delete`
  - Does NOT match patterns inside code comments or string literals within test assertions
  - Case-insensitive matching for SQL keywords (DROP TABLE, TRUNCATE)

  ## generateEndOfRunReport(cwd: string, milestone: string): string
  - Reads all SUMMARY.md files from .planning/phases/
  - Reads FAILURES.md if it exists
  - Reads all VERIFICATION.md files
  - Produces markdown report with sections: Summary table, What Was Built, What Was Verified, What Was Auto-Fixed, What Needs Human Attention
  - Summary table includes: phases completed, plans executed, auto-fixes applied, tasks needing attention
  - "What Was Auto-Fixed" populated from FAILURES.md resolved entries
  - "What Needs Human Attention" populated from FAILURES.md halted + skipped entries
